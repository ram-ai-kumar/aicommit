CONVENTIONAL COMMIT FORMAT:
<type>[optional scope][!]: <description>

[optional body]

[optional footer(s)]

STRUCTURE RULES:
1. Header: type(scope): description (max 50 chars, lowercase, no period, imperative mood)
2. Scope: Optional noun describing section of codebase (e.g., deps, auth, api, db)
3. Breaking: Use ! before colon for breaking changes: feat!: breaking change
4. Body: Detailed explanation (optional, starts after blank line, free-form)
5. Footers: Metadata after body (optional, format: token: value or token #value)
6. Footer tokens: MUST use - in place of whitespace (e.g., Acked-by, Reviewed-by)
7. Case: Types are case-insensitive. BREAKING CHANGE MUST always be uppercase.
8. BREAKING-CHANGE: Synonymous with BREAKING CHANGE when used as footer token.

BREAKING CHANGES:
- Method 1: Add ! before colon: feat!: breaking change description
- Method 2: Add BREAKING CHANGE: footer (MUST be uppercase)
- Method 3: Both ! and footer for emphasis
